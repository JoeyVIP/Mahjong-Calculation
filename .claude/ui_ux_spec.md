# UI/UX 規格

## 設計哲學

- **簡潔至上**：每個畫面只聚焦一個主要任務
- **觸控優先**：所有互動元素都為觸控優化
- **視覺回饋**：每個操作都有明確的視覺反應
- **現代美感**：使用當代設計趨勢，讓使用者感到愉悅
- **教育導向**：在使用中自然學會如何算台

## 色彩系統

### 主色調：
- **麻將桌綠**：#2D5F3F（深綠）到 #4A8B5C（中綠）的漸層
- **牌面白**：#FFFFFF
- **強調色**：#FFD700（金色，用於重要數字）
- **成功綠**：#10B981
- **警告橙**：#F59E0B
- **錯誤紅**：#EF4444

### 背景設計：
- 使用麻將桌綠色漸層背景
- 加入細微的紋理或圖案增加質感
- 不過度花俏，保持專業感

## 字體系統

- **中文**：思源黑體 或 系統預設 SF Pro Text / Noto Sans TC
- **數字**：SF Pro Display 或 Roboto（清晰易讀）
- **大標題**：48-64px，粗體
- **台數數字**：36-48px，粗體
- **說明文字**：14-16px，一般
- **小字**：12-14px

## 間距系統

- **基本單位**：8px
- **小間距**：8px
- **中間距**：16px
- **大間距**：24px
- **特大間距**：32px

## 按鈕尺寸

- **最小觸控區域**：60x60px
- **大按鈕**：80x80px（牌面選擇）
- **中按鈕**：60x44px（設定選項）
- **文字按鈕**：最小44px高度

## 頁面佈局（手機版，375x812px為基準）

### 結構：
- Header區（60px高）
- 設定面板（可摺疊，展開約200px）
- 手牌顯示區（120px）
- 選牌區（佔據剩餘空間，可滾動）
- 結果面板（浮動，從底部滑入）

### Header設計：
- 左側：Logo或標題「台麻計台器」
- 右側：設定icon（可展開設定面板）
- 使用漸層背景
- 有輕微陰影增加層次

### 設定面板設計：
- 預設摺疊，顯示當前設定摘要
- 點擊展開為全螢幕或半螢幕選擇器
- 大按鈕網格佈局
- **圈風**：四個大按鈕（東南西北）
- **座位**：四個大按鈕（東南西北）
- **莊家**：Toggle開關，大而清楚
- **自摸/胡別人**：兩個大按鈕切換
- **底台**：常用比例按鈕 + 自訂輸入
- 設定完成後自動摺疊

### 選牌區設計：
- **頂部**：分類Tab（萬、條、筒、字、花）
  - Tab使用大按鈕，當前分類有底色
- **牌面網格**：
  - 萬條筒：每排3個，每個80x80px
  - 字牌：每排4個
  - 花牌：每排4個
- **每張牌顯示**：
  - 清楚的牌面圖示
  - 右上角顯示已選數量（0-4）
  - 已選滿（4張）時變灰或顯示禁止icon
  - 點擊動畫：彈跳或放大效果
- 網格間距適中，避免誤觸

### 手牌顯示區設計：
- 固定在選牌區上方
- 白色背景，與選牌區區隔
- 顯示已選的牌，橫向排列
- 每張牌可點擊刪除（顯示小X）
- 右側顯示張數：12/17（大數字）
- 超過17張時張數變紅色並提示

### 結果面板設計：
- 手牌達17張時自動從底部滑入
- 佔據下半螢幕（約50-60%）
- 可上下拖曳調整大小或關閉
- 使用白色卡片，圓角，陰影

#### 總台數區塊（最突出）：
- 超大數字顯示總台數（金色，64px）
- 下方顯示「台」字
- 使用圖標或裝飾增加視覺吸引力

#### 台種列表：
- 每個台種一行
- 左側：台種名稱（加粗）
- 右側：台數（金色數字）
- 下方：簡短說明文字（灰色小字）
- 圈風台、門風台用icon標註
- 可展開查看詳細規則

#### 金額計算區塊：
- 顯著的區塊
- 公式展示：底50 + 台數5 × 20 = 150
- 自摸：每家付XXX（大數字）
- 胡別人：放槍者付XXX（大數字）
- 使用綠色表示收入

#### 操作按鈕區：
- 底部固定
- 清空按鈕：紅色或灰色，謹慎配色
- 返回修改按鈕（如果需要）

## 動畫規範

- **頁面進入**：淡入 + 輕微上移（300ms）
- **按鈕點擊**：縮小到0.95（100ms）
- **牌加入手牌**：彈跳動畫（spring效果）
- **牌移除**：縮小淡出（200ms）
- **結果面板出現**：從底部滑入（400ms，ease-out）
- **台數計數**：數字從0遞增到結果（800ms）
- **切換設定**：平滑過渡（200ms）

## 互動回饋

- **觸覺回饋**（如果裝置支援）：重要操作時震動
- **音效**（可選）：點擊、成功計算的音效
- **視覺**：
  - Hover（桌面）：輕微放大、陰影加深
  - Active：縮小、顏色加深
  - 選中狀態：邊框高光、背景色變化
  - 錯誤狀態：紅色邊框、搖晃動畫

## 響應式設計

### 手機（375px-768px）：
- 上述設計為基準
- 單欄佈局
- 按鈕較大，易於觸控

### 平板（768px-1024px）：
- 等比例放大
- 可考慮兩欄佈局（選牌+手牌/結果）
- 按鈕保持最小60x60px

### 桌面（1024px+）：
- 等比例放大或固定最大寬度（如600px居中）
- 可使用滑鼠Hover效果
- 按鈕可稍小但保持易點擊

## 可訪問性

- 顏色對比符合WCAG AA標準
- 重要按鈕有足夠的對比度
- 文字大小適中，易於閱讀
- 支援螢幕閱讀器（ARIA標籤）
- 支援鍵盤操作（Tab導航）
